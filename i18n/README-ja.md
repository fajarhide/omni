## 問題：コンテキストの肥大化、高価なトークン、ノイズの多い出力

自律型AIエージェント（Claude CodeやCursorなど）をターミナルで使用する場合、彼らは*すべて*を読み取ります。`git diff`、`npm install`、または`cargo test`などの単純なコマンドでも、AIのコンテキストに10,000〜25,000トークンの無駄なターミナルノイズを簡単にダンプする可能性があります。

これは3つの大きな問題を引き起こします。
1. **非常に高価**：ジャンク出力の1トークンごとにリアルマネーを支払うことになります。
2. **AIを「馬鹿」にする**：重大なエラーがメガバイト単位の警告ログやロードバーの下に埋もれ、AIを混乱させ、推論を希薄化させます。
3. **モデルのロックイン**：高度なエージェントフレームワークでは、すべてのノイズを処理するのに十分な大きさのコンテキストウィンドウを持つためだけに、最も高価なフラッグシップモデルの使用を強制されます。
4. **トークンを意識しない実行**：エージェントはトークンコストと出力を意識しないため、不必要な消費につながります。
5. **コンテキストの肥大化**：ターミナル出力の量がAIのコンテキストを圧迫し、集中力と精度を低下させます。

## 解決策：Omni

私は自分自身のワークフローで毎日、効率的かつ安価にAIエージェントを実行したかったため、Omniを構築しました。

**Omniは、ターミナルとAIの間の完璧なフィルターとして機能します。**

**結果は？** 超高度なフレームワーク上でAIエージェントを実行し、*ノイズゼロ*をフィードできます。AIは非常に焦点の絞られた要点をついたコンテキストのみをフィードされるため、手頃な価格のモデルや通常のモデルでも、ジャンクデータに気を取られることがないため、高価なフラッグシップモデルと同等のパフォーマンスを発揮します。

私の究極の情熱はこれを収益化することではありません。エージェンティックAI時代の究極のオープンソースツールベルトを構築することです。トークンコストを積極的に節約することで、私は今日堅牢かつ費用効果の高いソフトウェアを開発できますし、あなたもできます。

コンテキストは高価でノイズが多いですが、Omniはそれを修正するためにここにいます。コンテキストを最適化することで、OmniはAIエージェントをより効率的で、費用対効果が高く、使いやすくします。これは、AIエージェントに送信されるコンテキストの量を減らすことによって行われ、ひいては応答を生成するために必要な処理時間とメモリの量を減らします。

## 哲学

OMNIは単に「コンテキストをカットする」または「トークンを節約する」ためだけに構築されたわけではありません。それらは単なる嬉しい副産物です。OMNIの背後にある真の哲学は**コンテキストの品質**です。

ClaudeのようなAIエージェントは、あなたが提供するコンテキストの分だけ賢くなります。メガバイト単位の依存関係ログやロードバーで彼らを溢れさせると、実際の問題を見つけるためにゴミをふるいにかけることを強制することになります。これは彼らの推論を希薄化させ、劣化したまたは役に立たない応答につながります。

**OMNIの目標は、純粋で高密度なシグナルをAIに提供することです。** これは、Claudeにとって本当に重要で意味のあるコンテキストのみを取得することを意味します。AIが必要としないノイズをクリーンアップします。これは次のことを意味します。
1. 自動的に、使用するトークンが劇的に少なくなります。
2. コンテキストウィンドウが実際の問題にレーザーフォーカスされているため、AIの応答の**品質が大幅に向上**します。

**1週間試してみてください。** 生のターミナルノイズではなく、純粋なシグナルのダイエットを提供されたときの、AIの推論の品質と速度の違いを感じてください。

## 実際のユースケース

OMNIは、エージェンティックAI開発者の日常の不満を解決するために設計されています。OMNIがワークフローをどのように変革するかをご覧ください。

1. **モノレポにおける「無限ループの死」**
   - **シナリオ**: Claudeに大規模なモノレポで`npm install`と`npm run build`を実行するように依頼します。ターミナルは20,000行の依存関係の警告を出力し、最後に小さなビルドエラーを出力します。AIは警告に気を取られ、関係のない依存関係の問題を修正しようとし、トークンを消費して無限ループに陥ります。
   - **OMNIの解決策**: OMNIはビルドを傍受します。数百の`peer dependency`警告を完全にミュートし、スタックトレースと一緒に正確な`Build Error: Cannot find module 'X'`のみを表示します。AIは50トークンの出力を確認し、コードを即座に修正します。

2. **大きなファイルでの「静かなハルシネーション」**
   - **シナリオ**: AIはプロジェクトを理解したいと考え、`cat src/utils.ts`を実行します。ファイルは3,000行の長さです。AIはそれをすべてワーキングメモリに保持するのに苦労し、関数のシグネチャをハルシネーションし始めます。
   - **OMNIの解決策**: OMNIは生の`cat`をブロックし、それを**Structured Outline (構造化アウトライン)**に置き換えます。インポート、パブリックAPI（関数名とタイプ）、リスクマーカーをAIに表示し、出力を80%削減します。次に、OMNIはAIに警告します。`"このファイルには12の依存関係があります — 影響マップの全体像についてはomni_contextを使用してください。"` AIはより安全で事実に基づいた編集を行うように導かれます。

3. **マルチエージェントのコラボレーション**
   - **シナリオ**: クイック編集にCursor IDEを使用し、重い作業にClaude Code CLIを使用しています。冗長なコマンドを実行してトークンを無駄にすることなく、両方が何が起こっているかを知る必要があります。
   - **OMNIの解決策**: OMNIは共有メモリレイヤーとして機能します。`omni_agents`とそのローカルSQLite`Store`を使用して、CursorとClaudeは同じフィルタリングされたメモリ、アクティブなエラー、実行環境を共有します。競合することなくコラボレーションできます。

## 機能の解説

### コア蒸留エンジン (Core Distillation Engine)
- **AIの混乱を排除**: Omniはスマートなフィルターとして機能します。テストが失敗した場合、ノイズの多い依存関係のログをブロックし、特定のエラー行とスタックトレース*のみ*をAIに表示します。
- **トークンを90%削減**: 無駄なターミナルノイズを排除することで、エージェントのAPI料金を即座に大幅に削減します。
- **適応型圧縮 (Adaptive Compression)**: エージェントが省略された出力をいつ取得したかを追跡します。同じコマンドが頻繁に取得される場合、OMNIは次回から自動的に圧縮を緩めます（設定不要の自動調整）。
- **スマート高速バイパス**: 小さなタスクでの遅延をゼロにするため、2000トークン未満の出力に対しては自動的に蒸留をバイパスします。

### コンテキストの安全性と事実の保護 (Context Safety)
- **情報損失ゼロ**: 重要な情報がフィルタリングされる心配はありません。Omniは生の出力をローカル(`RewindStore`)に保存します。AIは`omni_retrieve`を使用して自動的に要求できます。
- **ハルシネーション防止ガード**: OMNIは確実な事実がある場合にのみ警告を発します。出力が大幅に圧縮された場合や、ファイルに巨大な依存関係がある場合、システム警告を注入してAIを現実に引き留めます。
- **省略の可視化**: 出力内の削除されたコンテンツを明示的にラベル付け（例：`[OMNI: omitted X lines of noise]`）し、AIエージェントに状況を正確に把握させます。

### マルチエージェントとワークスペースインテリジェンス
- **マルチエージェントのコラボレーション**: `omni_agents`を通じて環境を完全に把握。CursorとClaude CLIを同時に実行している場合でも、競合することなく同じフィルタリングされたメモリとエラーを共有します。
- **セッションインテリジェンス**: 編集中のファイルを記憶し、冗長なコンテキストの送信を停止します。修正は`omni_knowledge`を介して永続的に保存されます。
- **構造化されたReadFile + Grep**: 生のファイルダンプの代わりに、構造化されたアウトライン（インポート、パブリックAPI）とグループ化されたgrepサマリー（優先行を先頭に）を返します。
- **軽量の依存関係グラフ**: フック時に高速なローカルファイル関係グラフを構築します（デーモン不要）。AIが頻繁にインポートされるファイルを読み取った場合、影響マップについて警告します。

### コンテキストの忠実性とセッションの回復 (Context Fidelity & Session Recovery)
- **エングラム (自動サブタスク要約)**: OMNIは、サブタスク（コンパイルエラーの解決、コードのコミット、失敗したテストの修正など）が完了したことを自動的に検出します。LLM呼び出しでトークンを無駄にすることなく、高度に圧縮されたスナップショット（「エングラム」）を作成するため、長時間のセッション中にエージェントが「コンテキスト健忘症」に陥ることはありません。
- **スマートコンテキスト圧縮 (Smart Context Compaction)**: コンテキストウィンドウがいっぱいになると、OMNIはむやみにトークンをトリミングしません。優先順位を認識するアルゴリズムを使用して、最も重要なデータ（ピン留めされたファイル > アクティブなエラー > エングラム > ツールの非アクティブ > ホットファイル）を最初にパックし、大規模なオーバーヘッドを節約します。
- **セッションの引き継ぎ (Session Handoffs)**: Claude CodeからCursorに切り替えますか？ `omni_handoff` ツールを使用して、現在のセッションのメモリ（ホットファイル、最近のコマンド、アクティブなエラー）を、新しいエージェントがすぐに吸収できるポータブルなマークダウンサマリーに即座にエクスポートします。

### ループエンジニアリング (Autonomous Loop Engineering)
- **ループのためのコンテキストOS**: OMNIは、反復的な自律ループエージェントのコンテキストを管理します。環境変数（`OMNI_LOOP_BUDGET`、`OMNI_LOOP_GOAL`）を通じて、OMNIは適応的な蒸留制限と永続的な追跡を強制します。
- **メーカー-チェッカー検証パターン**: タスクの実行（メーカー/Makerエージェント）から検証（チェッカー/Checkerエージェント）を分離し、OMNIのマルチエージェントセッションストレージを介して安全にコンテキスト状態を交換することで、タスクを効率的にスケーリングします。
- **予測可能な目標ベースの制限**: 蒸留は、タスクの目標に基づいて自動的にスケーリングされます。目標に "debug" が含まれている場合、OMNIはより多くのエラーコンテキストを保持します。 "refactor" の場合、OMNIはコードトレースを積極的に圧縮します。

### モニタリングとデバッグ
- **セッションヘルスダッシュボード**: `omni session --health`を実行すると、コンテキストのプレッシャー、アクティブなエングラム、ローリングツールアクティビティ、およびトークンの節約を視覚的に表示する美しいダッシュボードが表示されます。
- **蒸留モニター**: `omni_budget`と`omni_history`をLLM内で直接使用するか、ローカルで`omni stats`を実行して節約したトークンを追跡します。
- **視覚的なインパクト (`omni diff`)**: `omni diff`を実行して、生の出力とOmniのフィルタリングされたバージョンを横に並べて比較できます。
- **デバッグパススルー**: 生の出力が必要な場合は、環境変数`OMNI_PASSTHROUGH=1`を設定してエンジンを完全にバイパスし、元の出力のすべての文字を確認できます。

## Omniの仕組みの裏側

OMNIは単なる正規表現スクリプトではありません。Rustで記述された高性能な**セマンティックシグナルエンジン**です。しかし、どのようにして100ミリ秒未満でトークン消費の90%を削減するのでしょうか？

AIエージェントが`cargo test`のようなコマンドを入力したときにOMNIコードベース内で何が起こるかについてのストーリーです。

1. **傍受 (`src/hooks` & `src/main.rs`)**: AIが「Enter」を押した瞬間、OMNIは実行を傍受します。`main.rs`はコンテキスト（パイプか、フックか、MCP呼び出しなど）を動的に検出します。`hooks`モジュールはコマンドをシームレスにラップし、OMNIが実際の実行を遅らせることなく、高速データストリームとして生のターミナル出力をキャプチャできるようにします。
2. **ストリーミングパイプライン (`src/pipeline`)**: コマンドが終了してメガバイトのテキストをメモリにダンプするのを待つのではなく、OMNIはメモリ効率の高いストリーミングパイプラインを使用して行ごとに出力を処理します。これにより、コマンドが10,000行のログを吐き出した場合でも、OMNIのメモリフットプリントはほぼフラットなままになります。
3. **セマンティックブレイン (`src/distillers` & `src/guard`)**: テキストがストリーム入力されると、ディスティラーを通過します。宣言型TOMLルール（`signals/`）を搭載したディスティラーは、出力の意味を分析します。
   - これはロードスピナーですか？ *破棄します。*
   - これは合格した500のテストのリストですか？ *破棄します。*
   - これはパニックスタックトレースですか？ **保持します。**
   一方、`guard`モジュールは事実が保持されることを保証し、OMNIが重要な診断情報を密かに変更しないことを保証します。
4. **セーフティネット (`src/store`)**: AIが実際に合格した500のテストを見る必要があった場合はどうなりますか？ OMNIは厳密な「情報損失ゼロ」ポリシーに従います。ノイズが破棄される前に、編集されていない生の出力は、ローカルの超高速SQLiteデータベース（`Store`）に安全に収納されます。OMNIはAIのコンテキストに小さなブレッドクラムを残します。`[OMNI: omitted 1,200 lines of noise. Use omni_retrieve to view]`
5. **マルチエージェントインターフェース (`src/mcp` & `src/session`)**: 最後に、蒸留された高シグナル出力がAIに返されます。舞台裏では、`session`マネージャーが現在のトークン予算を追跡し、`mcp` (Model Context Protocol) サーバーが準備を整えています。AIが履歴エラーをクエリしたり、省略された生のログを取得したり、依存関係グラフ（`src/graph`）を確認したりしたい場合、MCPツールは即座に構造化されたアクセスを提供します。

**結果:** 肥大化した`25,000`トークンのターミナルダンプは、簡潔な`400`トークンのエラーレポートになります。AIは問題を即座に理解し、あなたはリアルマネーを節約します。

## クイックスタートとインストール

Omniのセットアップは非常に簡単です。ターミナルにネイティブに統合されます。

**macOS / Linux:**
```bash
# 1. Homebrew経由でインストール
brew install fajarhide/tap/omni

# 2. Omniのセットアップ (Claude、VS Code、OpenCode、Codex、Antigravityのインタラクティブメニュー)
omni init

# 3. 正常に動作しているか確認
omni doctor

# 4. または問題を自動修正
omni doctor --fix

# 5. 現在のステータスを確認
omni init --status
```

## 使い方

`omni init`を介してインストールされると、OMNIはバックグラウンドで見えないように機能します。AIエージェントがMCPを介してターミナルコマンドを実行するか、手動で出力をパイプする（`ls | omni`）かにかかわらず、OMNIは透過的なレイヤーとして自動的に飛び込みます。ターミナル出力をインテリジェントにフィルタリングし、ノイズの多いログを削除して、クリーンなシグナルをAIに返します。

節約量、コマンド、期間、ルートごとの詳細な内訳を見るには：
```bash
omni stats
```

OMNIのインストール（フック、MCP、フィルター、データベース）を診断するには：
```bash
omni doctor
```

フィルターが動作しているのを確認したり、独自のカスタムルールを追加したりする必要がありますか？
`~/.omni/signals/`にあるシンプルなTOMLファイルを使用して、独自のルールを簡単に作成できます。

### マルチエージェントのサポートと統合

デフォルトでは、`omni init --claude`は自動的に**Claude Code**にフックします。ただし、OMNIは組み込みの統合を通じて、あらゆるエージェンティックAIと完全に機能します！ `omni init`を実行してインタラクティブメニューを表示します。

1. **VS Code & Continue.dev**: MCPコンテキストプロバイダー（`integrations/continue-dev/`）を使用します。
2. **OpenCode & Codex CLI**: 組み込みのラッパーがコマンド出力を自動的にOMNIにパイプします。
3. **Antigravity IDE**: OMNIはAntigravityの設定（`~/.gemini/antigravity/mcp_config.json`）でネイティブMCPサーバーとして登録します。自動的にセットアップするには`omni init --antigravity`を実行します。
4. **Pi Agent**: Pi用のネイティブOMNIパッケージ。Piのパッケージインストーラー経由でインストールするには、`omni init --pi`を実行します。

**マルチエージェントのチューニング (`~/.omni/config.toml`)**
エージェントによってペインポイントは異なります。VS Codeのチャットをクリーンに保ちながら、OpenCodeにより多くのデータを読み取らせます。それらを個別に調整します。
```toml
[global]
aggressiveness = "balanced"

[agents.vscode_continue]
aggressiveness = "aggressive"
enable_readfile_distillation = true

[agents.opencode]
aggressiveness = "conservative"
enable_readfile_distillation = false
```

## Heimsenseとの連携

Omniは私の個人的なAIツールベルトの一部です。`claude-code`を使用している場合は、Omniを私のもう1つのプロジェクトである**[Heimsense](https://github.com/fajarhide/heimsense)**とペアリングすることを強くお勧めします。

Heimsenseは、高価なAnthropicモデルの使用を強制するのではなく、`claude-code`のような制限された環境のロックを解除して、*任意の*無料またはOpenAI互換モデルで実行できるようにします。
**Omni + Heimsense** = 手頃な価格のモデルを使用して、ノイズゼロで正確なワールドクラスのエージェントフレームワークを実行します。
